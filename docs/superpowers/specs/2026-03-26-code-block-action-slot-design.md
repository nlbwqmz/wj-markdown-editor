# 代码块右上角单槽位交互设计

## 1. 背景

当前普通 fenced code block 已经收口到以下主链路：

- 渲染结构由 `wj-markdown-editor-web/src/util/markdown-it/markdownItCodeBlock.js` 输出。
- 结构层样式由 `wj-markdown-editor-web/src/assets/style/code-block/code-block-base.scss` 承接。
- 代码主题由 `wj-markdown-editor-web/src/util/codeThemeUtil.js` 动态注入，并通过 `code-theme-${codeTheme}` 挂到预览根节点上。

前一轮设计已经明确了代码块职责边界：

1. preview theme 只负责正文排版与 inline code。
2. code block base 负责 fenced code block 的结构、交互与中立几何。
3. code theme 负责 `.hljs` 皮肤与 token 颜色。

本轮需求不是重新划分这三层职责，而是在这个边界已经成立的前提下，进一步修正代码块右上角控件的观感与交互。

当前用户明确指出的问题是：

- 语言标记与复制按钮虽然功能上可用，但视觉上像“漂浮在代码块之外”。
- 复制按钮常驻会让工具栏存在感过强。
- 右上角控件颜色容易和代码主题背景冲突。
- 长代码、长语言标签和右上角控件之间缺少明确避让关系。

因此，本轮设计目标是把右上角控件收回到代码块容器内部，并把语言标记与复制按钮收口成一个单槽位交互。

## 2. 已确认约束

用户已经明确确认以下约束，本轮设计必须全部满足：

1. 在当前分支完成，不新建分支或 worktree。
2. 执行任务时继续使用 subagent。
3. 语言标记位于代码块右侧，视觉上贴近右上角。
4. 默认常驻显示语言标记。
5. 鼠标 hover 代码块时，语言标记隐藏，复制按钮在同一位置显示。
6. 语言标记与复制按钮不是并排共存，而是同槽位来回切换。
7. 不拆“显示标签”和“高亮语言”，语言标记继续显示用户原始 fence 文本。
8. 不做紧凑语言别名映射，不把 `javascript` 压缩成 `js`，也不把 `js` 擅自扩展成 `javascript`。
9. 右上角控件颜色不能与代码块背景冲突。
10. 长代码时右上角控件与正文不能互相遮挡。
11. 顶部需要安全区，但安全区限制的是高度，不是宽度，而且高度要尽量薄。
12. 导出页不增加特殊逻辑分支；由于没有 hover，复制按钮自然不会显示。

## 3. 目标

本轮设计目标如下：

1. 让语言标记与复制按钮都成为代码块容器内部的一部分，不再表现为独立工具栏。
2. 把右上角控件重构为“单槽位切换”模型，默认语言标记，交互时切换为复制按钮。
3. 在不改动 72 个代码主题源码的前提下，让语言标记与复制按钮颜色自动跟随当前代码主题，避免撞色。
4. 建立“薄顶部安全带 + 同槽位浮层”的避让模型，保证长代码与右上角控件不互相压住。
5. 调整现有测试边界，把旧的“两行 toolbar”假设替换成新的内嵌单槽位假设。

## 4. 非目标

本轮明确不做以下事项：

1. 不重做 markdown-it 的 fenced code block 主渲染链路。
2. 不修改 Mermaid 代码块分支的结构与外观。
3. 不新增或修改 72 个代码主题文件中的配色规则。
4. 不把导出页拆成单独的代码块样式实现。
5. 不新增第二个右上角按钮，也不引入更多 hover 工具项。
6. 不在本轮更改 `window.copyCode` 的调用契约。
7. 不改变当前显式语言、自动识别语言和隐藏语言标签的已有语义规则。

## 5. 当前实现与问题定位

### 5.1 当前结构

当前普通 fenced code block 由 `markdownItCodeBlock.js` 输出：

```html
<div class="pre-container">
  <div class="pre-container-toolbar">
    <div class="pre-container-lang">js</div>
    <div class="pre-container-copy" role="button" tabindex="0" ...></div>
  </div>
  <pre>
    <code class="hljs language-js language-javascript">...</code>
  </pre>
</div>
```

### 5.2 当前样式问题

`code-block-base.scss` 当前仍采用“两行 grid”布局：

- `.pre-container-toolbar` 占据第 1 行。
- `pre` 占据第 2 行。

这会带来两个直接问题：

1. 工具栏天然像“正文上面额外插了一行”，不够像代码块内部的一部分。
2. 复制按钮和语言标记处于一个独立行上下文中，无法自然完成“同一位置切换”。

### 5.3 当前配色问题

`markdownItCodeBlock.js` 现在把语言标记和复制按钮都写成固定的正文语义色：

- `color-[var(--wj-markdown-text-secondary)]`

这套颜色来源不跟代码主题绑定，在浅底、深底、纹理底和高饱和底色下都会出现以下风险：

1. 图标和标签对比度不足。
2. 右上角控件看起来像浮在不属于它的背景上。
3. 不同代码主题切换时，右上角控件缺乏一致的视觉归属感。

### 5.4 当前测试问题

`wj-markdown-editor-web/src/assets/style/__tests__/codeBlockThemeBoundary.test.js` 里仍保留了以下旧假设：

1. `.pre-container` 必须是两段式 grid。
2. `.pre-container-toolbar` 不能 absolute。
3. 不允许通过顶部安全区让位。

这几条断言与本轮目标正面冲突，若不改写测试，后续实现即使正确也会被错误阻断。

## 6. 设计结论

### 6.1 总体方案

本轮采用“右上角单槽位切换”方案：

1. 继续保留 `.pre-container`、`.pre-container-toolbar`、`.pre-container-lang`、`.pre-container-copy` 这套主命名，避免大范围改动渲染与测试链路。
2. 在 toolbar 内新增一个单槽位包装节点，例如 `.pre-container-action-slot`。
3. `.pre-container-toolbar` 不再作为独立工具栏行存在，而是作为代码块容器内部右上角浮层存在。
4. 默认只显示 `.pre-container-lang`。
5. 当 `.pre-container` 进入 `:hover` 或 `:focus-within` 状态时，`.pre-container-lang` 隐藏，`.pre-container-copy` 在同一位置显示。

### 6.2 目标 DOM

目标结构建议调整为：

```html
<div class="pre-container">
  <div class="pre-container-toolbar">
    <div class="pre-container-action-slot">
      <div class="pre-container-lang">javascript</div>
      <div
        class="pre-container-copy"
        role="button"
        tabindex="0"
        title="复制"
        aria-label="复制"
      ></div>
    </div>
  </div>
  <pre>
    <code class="hljs language-javascript">...</code>
  </pre>
</div>
```

其中需要注意：

1. `pre-container-action-slot` 是本轮新增的唯一结构包装节点。
2. 语言标记和复制按钮都位于这个包装节点内，后续交互只在此槽位中切换。
3. 复制按钮及其祖先链路不得使用 `aria-hidden="true"`，避免把按钮移出可访问树。
4. `pre-container-toolbar` 仅承担视觉定位职责，不承担可访问性裁剪职责。

### 6.3 交互规则

交互规则固定如下：

#### 默认态

- 显示语言标记。
- 复制按钮不可见、不可点击。
- 右上角控件仍保留在容器内部对应槽位中。

#### Hover 态

- 当鼠标 hover `.pre-container` 时，语言标记隐藏。
- 复制按钮在同一位置显示。
- 不额外增加第二个槽位，不做左右并排布局。

#### 键盘焦点态

- 当复制按钮获取焦点，或 `.pre-container` 进入 `:focus-within`，应与 hover 态保持一致。
- 即使没有鼠标 hover，只要键盘焦点进入复制按钮，语言标记也应切换隐藏，复制按钮保持可见。

#### 导出页

- 不新增导出页专属逻辑。
- 导出页由于不存在 hover 交互，默认只显示语言标记。
- 这属于同一套样式在不同交互环境下的自然结果，而不是专门为导出页分支。

## 7. 布局与避让方案

### 7.1 容器模型

布局从“两行 grid”改为“单容器 + 内部浮层”：

1. `.pre-container` 成为单一结构容器。
2. `.pre-container-toolbar` 改为容器内部右上角定位层。
3. `pre` 继续承接正文滚动，不改变其作为代码块主滚动层的职责。

推荐几何关系如下：

1. `.pre-container` 使用 `position: relative`。
2. `.pre-container-toolbar` 使用 `position: absolute` 锚定到右上角。
3. `.pre-container-action-slot` 控制内容的最小点击区域与视觉外壳。

### 7.2 安全区模型

本轮采用“薄顶部安全带”而不是“右侧宽安全区”。

具体含义如下：

1. 不为右上角控件额外保留一整块固定宽度区域。
2. 只利用代码块顶部已有的内边距，形成一条很薄的垂直安全带。
3. 语言标记和复制按钮都停靠在这条薄安全带内。

这样做有三个直接收益：

1. 满足“安全区不要太大”的要求，特别是高度不再像工具栏那样显眼。
2. 不挤压正文横向可视区域，长代码仍以代码块本身滚动为主。
3. 语言标记宽度可以自然增长，不被预先切成很窄的按钮宽度。

### 7.3 与现有代码主题 padding 的关系

对现有 72 个代码主题的静态扫描结果表明，`pre code.hljs` 全部统一声明了 `padding: 1em`。

这意味着：

1. 代码块顶部本来就存在稳定的内部空间。
2. 右上角槽位可以直接落在这条现有空间内，而不必再人为新增一整行 toolbar。
3. 本轮只需要对顶部安全带做很小幅度的结构补偿，不需要修改每个主题文件。

### 7.4 长代码与长标签的避让规则

为避免语言标记和长代码、长标签之间互相遮挡，本轮固定如下规则：

1. 正文不围绕右上角控件做复杂文本回流，避免引入新的多主题不一致。
2. 右上角控件只占用顶部薄安全带，不压在正文第一行主阅读高度上。
3. 语言标记允许按原始文本自然展开，但必须限制为单行。
4. 当语言文本超出合理宽度时，只允许单行截断，不允许换行增高。
5. 复制按钮在同一槽位中显示，因此不会额外扩大右上角占位宽度。

这意味着安全区控制重点在高度而不在宽度，符合用户已确认的要求。

## 8. 颜色策略

### 8.1 基本原则

右上角控件颜色必须跟随当前代码主题，而不是复用正文主题色。

本轮明确采用“运行时采样 + 结构层变量”的颜色策略：

1. 不修改 72 个代码主题源码。
2. 不在 `code-block-base.scss` 中直接写 `.code-theme-*` 分支。
3. 在代码主题样式已加载、预览 DOM 已刷新之后，读取当前 `.hljs` 的真实计算样式。
4. 将采样结果写入结构层 CSS 变量，由 `code-block-base.scss` 消费。

### 8.2 采样来源

运行时至少采集以下信息：

1. `.hljs` 的 `color`
2. `.hljs` 的 `background-color`
3. 必要时补充读取 `background-image`

原因如下：

1. `.hljs` 的真实文字色最接近当前代码主题的主前景色。
2. `.hljs` 的真实背景色决定右上角控件是否容易融到底色里。
3. 个别主题存在纹理或非纯色背景，单纯取文本色不足以构造可读控件。

### 8.3 建议变量

运行时建议派生并写入以下变量：

1. `--wj-code-block-action-fg`
2. `--wj-code-block-action-fg-muted`
3. `--wj-code-block-action-bg`
4. `--wj-code-block-action-border`
5. `--wj-code-block-action-shadow`

消费关系如下：

- 语言标记默认使用 `fg-muted`
- 复制按钮显示时使用 `fg`
- 单槽位实体背景使用 `action-bg`
- 细边框与弱阴影分别使用 `action-border`、`action-shadow`

### 8.4 控件形态

本轮不建议让语言标记和复制按钮直接“裸贴”在代码块背景上，而应保持一个很轻的实体槽位。

原因如下：

1. 仅仅换图标颜色，仍然可能在复杂背景中失去边界。
2. 语言标记与复制按钮需要共享同一位置，实体槽位更容易建立归属感。
3. 实体槽位可以非常轻，不需要做成明显胶囊，只需要足够让控件从代码背景中略微分离。

### 8.5 回退策略

运行时取色需要提供回退逻辑，避免采样失败时出现不可见控件。

最少需要以下回退规则：

1. 若 `.hljs` 前景色读取失败，则回退到高对比中性色。
2. 若背景色透明或不可用，则回退到当前 `.hljs` 背景与前景构造出的半透明实体底色。
3. 若最终对比度不足，则根据背景明暗回退为深色或浅色图标。

该回退策略只影响结构层变量，不应回流到任何 theme 文件。

## 9. 可访问性与显隐策略

### 9.1 复制按钮保留在 DOM 中

复制按钮必须始终保留在 DOM 中，不允许在默认态直接移除。

原因如下：

1. 若使用 `display: none`，键盘焦点无法进入该按钮。
2. 若按钮完全从可访问树消失，则 `focus-within` 也无法触发切换。

### 9.2 建议的显隐方式

建议采用以下组合控制：

1. `opacity`
2. `pointer-events`
3. `transform` 或轻微位移动画（可选）

其中需要明确：

1. 复制按钮默认态可以视觉隐藏，但必须保持可聚焦。
2. 复制按钮及其祖先不得使用 `display: none`、`visibility: hidden`、`hidden` 或 `aria-hidden="true"`。
3. 若需要阻断鼠标点击，只能通过 `pointer-events` 控制，不得破坏键盘聚焦路径。

默认态下：

- 语言标记可见、可读。
- 复制按钮透明且不接收鼠标事件，但仍保留在顺序焦点路径中。

hover / focus-within 态下：

- 语言标记透明并失活。
- 复制按钮可见并恢复可点击。

### 9.3 焦点可见性

复制按钮在键盘焦点态下应有稳定的 focus ring 或等价高对比反馈，且该反馈颜色同样来自结构层变量。

这条规则的重点不是视觉装饰，而是保证：

1. 用户能明确知道当前焦点是否在复制按钮上。
2. 深浅主题切换后，焦点态仍然不会丢失。

## 10. 文件职责调整

### 10.1 `markdownItCodeBlock.js`

本文件只做以下最小结构调整：

1. 在 toolbar 内增加单槽位包装节点。
2. 保持语言标签与复制按钮的现有语义节点不变。
3. 继续维持普通 fenced code block 与 Mermaid code block 的分支分离。

本文件不负责：

1. 主题颜色计算。
2. 主题特定样式分支。
3. 导出页特判。

### 10.2 `code-block-base.scss`

本文件成为本轮的主要样式承接点，负责：

1. 单容器内部浮层布局。
2. 顶部薄安全带与右上角停靠关系。
3. 单槽位切换规则。
4. 语言标记与复制按钮的基础几何与显隐。
5. 结构层变量消费。

本文件继续禁止：

1. `.code-theme-*` 分支。
2. `.hljs-*` token 颜色规则。
3. 预览主题人格化规则。

### 10.3 `MarkdownPreview.vue`

本文件负责运行时采样与结构层变量注入：

1. 在代码主题加载完成并预览刷新完成后读取 `.hljs` 计算样式。
2. 将结果写入 `MarkdownPreview.vue` 当前预览根节点，不写入全局作用域。
3. 与现有 `codeThemeUtil.js` 动态注入链路保持顺序一致。

### 10.4 `codeThemeUtil.js`

如果运行时采样逻辑更适合放在主题加载后链路中，也可以让 `codeThemeUtil.js` 协助触发变量刷新，但它不应承担具体 DOM 结构逻辑。

## 11. 测试调整方案

### 11.1 `markdownItCodeBlock.test.js`

需要继续保留以下旧边界：

1. 普通 fenced code block 必须输出语言标签节点。
2. 复制按钮仍然必须具备 `role="button"`、`tabindex="0"`、`title="复制"`、`aria-label="复制"` 等可访问性契约。
3. Mermaid 分支仍然不得输出 toolbar 与 copy 结构。

需要新增或修改以下断言：

1. 允许 toolbar 中出现 `pre-container-action-slot` 包装节点。
2. 语言标签与复制按钮位于同一槽位容器内。
3. 不再把“两个节点并排存在于同一工具栏行”当作唯一结构假设。

### 11.2 `codeBlockThemeBoundary.test.js`

需要删除或重写以下旧假设：

1. `.pre-container` 必须采用两行 grid。
2. `.pre-container-toolbar` 不得 absolute。
3. 不得通过顶部安全带让位。
4. 正文必须固定落在 toolbar 结构行之后。

需要改成的新边界：

1. `.pre-container` 必须继续是结构容器。
2. `.pre-container-toolbar` 必须继续限定在 `.wj-preview-theme` 作用域内。
3. 结构层规则仍然只能放在 `code-block-base.scss`。
4. `code-block-base.scss` 仍然不得出现 `.code-theme-*` 分支与 token 颜色规则。
5. 预览主题文件仍然不得命中 `.pre-container*`、`.hljs*`、`pre.mermaid*` 等结构禁区。

### 11.3 运行时取色测试

若本轮为取色逻辑抽出独立函数，应补以下测试：

1. 能从合法的 `.hljs` 前景与背景值生成结构层变量。
2. 当前景、背景或背景图采样失败时能回退到安全值。
3. 主题切换后变量会刷新，而不是停留在旧主题。

### 11.4 人工回归

人工回归至少覆盖：

1. 深色代码主题与浅色代码主题各若干个。
2. 默认预览页、编辑页预览区、导出页。
3. 显式语言、自动识别语言、未识别语言、无语言、长代码、长语言标签。

重点观察：

1. 默认态是否只显示语言标记。
2. hover 时是否稳定切成复制按钮。
3. 复制图标颜色是否与代码主题背景有足够对比。
4. 长代码首行是否被右上角控件压住。
5. 导出页是否自然只保留语言标记。

## 12. 风险与控制

### 12.1 风险：主题采样时机错误

若在代码主题尚未注入完成或预览 DOM 尚未刷新完成时采样，将会得到旧主题颜色或空值。

控制方式：

1. 取色必须发生在 `loadCodeTheme()` 完成之后。
2. 同时应发生在预览 DOM 已更新之后。
3. 若主题切换和内容刷新交叉发生，以最后一次渲染完成结果为准。

### 12.2 风险：复杂背景主题下控件仍然不够清晰

个别主题可能带纹理、渐变或对比极低背景。

控制方式：

1. 右上角控件保留轻实体槽位，不使用裸图标。
2. 必要时引入轻边框或轻阴影。
3. 统一通过运行时变量派生控制，而不是新增主题特例。

### 12.3 风险：键盘可达性被 hover 方案破坏

若复制按钮默认完全隐藏或不可聚焦，会导致键盘用户无法使用复制功能。

控制方式：

1. 不使用 `display: none` 作为主要隐藏手段。
2. 使用 `focus-within` 与 hover 共享切换规则。
3. 在测试中继续锁定 `role`、`tabindex`、`aria-label` 等基础契约。

### 12.4 风险：长语言标签把控件高度撑大

若语言标签允许换行，会直接破坏“薄顶部安全带”约束。

控制方式：

1. 语言标签固定单行。
2. 超出时只做截断，不允许换行。
3. 单槽位最小高度应与复制按钮高度对齐，不因文案长度改变。

## 13. 实施顺序

建议按以下顺序实施：

1. 在 `markdownItCodeBlock.js` 中补充单槽位包装节点。
2. 在 `code-block-base.scss` 中把两行 toolbar 模型改为单容器内部浮层模型。
3. 把默认态、hover 态、focus-within 态的语言标记 / 复制按钮切换规则写入结构层。
4. 在 `MarkdownPreview.vue` 或合适的运行时层补充 `.hljs` 取色与结构层变量写入。
5. 调整 `markdownItCodeBlock.test.js` 的 DOM 契约断言。
6. 调整 `codeBlockThemeBoundary.test.js` 的结构边界断言。
7. 做深浅主题、长代码、长语言标签与导出页回归验证。

## 14. 完成标准

当满足以下条件时，本轮设计目标视为达成：

1. 语言标记和复制按钮都位于代码块内部右上角，不再表现为独立工具栏行。
2. 默认只显示语言标记，hover 或 focus-within 时在同一位置切成复制按钮。
3. 语言标记继续显示原始语言文本，不拆标签语义，也不做紧凑别名。
4. 复制图标与语言标记颜色能够跟随当前代码主题，不与背景冲突。
5. 顶部安全区足够薄，长代码与右上角控件不互相遮挡。
6. 导出页无需特判即可自然只显示语言标记。
7. Mermaid 分支行为不变。
8. 自动化测试已从旧的“两行 toolbar”假设切换为新的“单槽位内嵌浮层”假设，并能持续防止职责回流。
