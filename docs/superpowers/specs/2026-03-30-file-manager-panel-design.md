# 主窗口文件管理栏设计

## 1. 背景

当前主窗口已经稳定收口为两层结构：

- 主窗口壳层由 [HomeView](../../../wj-markdown-editor-web/src/views/HomeView.vue) 负责，内部承载编辑页与预览页。
- 文档真相与窗口生命周期由 Electron 侧 `document-session` 主线统一裁决，不再允许 renderer 侧绕开 runtime 自行切换文档状态。

本次需求希望在进入主窗口后，于最左侧新增一个文件管理视图，并支持：

- 默认定位到当前文件目录
- 仅监听当前目录一层文件变化
- 新建文件夹与新建 Markdown 文件
- 点击目录进入该目录
- 高亮当前文件
- 点击 Markdown 文件时，允许用户选择在当前窗口打开或在新窗口打开
- 当前窗口打开时，若当前文档未保存，需要先做保存决策
- 文件管理栏支持显隐和拖动调整宽度，且需要限制最大宽度
- 顶部菜单“最近历史”点击打开时，也复用同一套“当前窗口 / 新窗口”选择逻辑

## 2. 目标与非目标

### 2.1 目标

1. 在编辑页与预览页提供稳定的左侧文件管理栏，不影响其他页面。
2. 文件管理栏只展示当前目录一层内容，不递归展示目录树。
3. 文件管理栏列表区域只在侧栏内部滚动，不给主窗口引入全局滚动条。
4. 当前窗口打开目标 Markdown 时，严格走主进程统一切换命令，正确完成 session 重绑、watcher 重绑与快照广播。
5. 顶部“最近历史”和左侧文件管理栏复用同一套打开决策与确认流程。

### 2.2 非目标

1. 本次不实现递归文件树。
2. 本次不实现非 Markdown 文件的预览、系统打开或拖拽导入。
3. 本次不引入多标签模型。
4. 本次不改变 `document.open-path` 现有“新建窗口或聚焦已有窗口”的既定语义。

## 3. 适用范围

文件管理栏只在以下页面显示：

- [EditorView](../../../wj-markdown-editor-web/src/views/EditorView.vue)
- [PreviewView](../../../wj-markdown-editor-web/src/views/PreviewView.vue)

文件管理栏不出现在以下页面：

- [SettingView](../../../wj-markdown-editor-web/src/views/SettingView.vue)
- [AboutView](../../../wj-markdown-editor-web/src/views/AboutView.vue)
- [GuideView](../../../wj-markdown-editor-web/src/views/GuideView.vue)
- [ExportView](../../../wj-markdown-editor-web/src/views/ExportView.vue)

因此文件管理栏应挂载在 [HomeView](../../../wj-markdown-editor-web/src/views/HomeView.vue) 的内容区左侧，而不是进入 [MarkdownEdit](../../../wj-markdown-editor-web/src/components/editor/MarkdownEdit.vue) 内部布局。

## 4. 交互设计

### 4.1 主窗口布局

主窗口内容区调整为三列结构：

- `file-panel`
- `gutter-file-panel`
- `content`

其中：

- `content` 继续承载现有 [LayoutContainer](../../../wj-markdown-editor-web/src/components/layout/LayoutContainer.vue)。
- 文件管理栏与主内容区之间使用单独的 `split-grid` 分隔条。
- 文件管理栏支持显隐、拖动调宽，但不影响编辑器内部已有的 preview/menu 分隔条逻辑。

### 4.2 文件管理栏宽度规则

- 默认宽度：`260px`
- 最小宽度：`200px`
- 最大宽度：`420px`

设计理由：

- `200px` 能保证图标、文件名和创建按钮不发生严重挤压。
- `420px` 足够覆盖较长文件名与面包屑，同时避免左栏过宽压缩编辑工作区。

### 4.3 滚动与布局约束

- 主窗口内容区继续保持 `overflow-hidden`。
- 文件管理栏外层使用 `overflow-hidden`。
- 文件列表区使用 `overflow-y-auto` 且 `min-height: 0`。
- 当文件数量较多时，只允许文件管理栏内部出现滚动条，不允许主窗口出现新的全局滚动条。

### 4.4 文件管理栏内部结构

文件管理栏拆分为两个主要区域：

1. 顶部工具区
   - 显示当前目录标题或面包屑
   - 提供“新建文件夹”“新建 Markdown”操作
2. 中间列表区
   - 独立滚动
   - 展示当前目录下的所有一级条目

首版不单独增加底部状态栏，状态提示通过空状态或消息提示完成。

### 4.5 列表展示规则

- 当前目录下的所有一级条目都展示，不过滤文件类型。
- 目录排在前面，文件排在后面。
- 同类型内按名称排序。
- 当前文档对应的文件行高亮。
- 文件名过长时单行省略，不撑高列表项。
- 目录、Markdown 文件、其他文件使用不同图标表达。

### 4.6 三种初始态

#### 正常文件会话

- 默认定位到当前文件所在目录。
- 在列表中高亮当前文件。

#### `draft`

- 文件管理栏显示空状态。
- 不自动绑定任何默认目录。
- 空状态中提供后续动作入口，例如“打开文件夹”。

#### `recent-missing`

- 如果缺失文件原父目录仍存在，则定位到原父目录。
- 因当前文件已不存在于目录中，所以列表中没有高亮项。
- 如果原父目录也不存在，则文件管理栏直接进入空状态。
- 当 `recent-missing` 的父目录不存在时，文件管理栏不额外展示文案提示。

### 4.7 点击行为

#### 点击目录

- 进入该目录。
- 文件管理栏当前监听目录切换到新目录。
- 列表刷新为新目录的一层内容。

#### 点击当前文档对应的 Markdown 文件

- 不执行任何操作。

#### 点击其他 Markdown 文件

进入统一打开决策流程：

1. 先询问“在当前窗口打开 / 在新窗口打开 / 取消”
2. 若选择“新窗口打开”，走现有 `document.open-path`
3. 若选择“当前窗口打开”，走新增 `document.open-path-in-current-window`
4. 若当前文档未保存，再插入“先保存再切换 / 不保存直接切换 / 取消”

#### 点击其他文件类型

- 首版仅提示“不支持的文件类型”
- 不执行系统打开
- 不执行预览
- 不执行切换

### 4.8 新建操作

#### 新建文件夹

- 在当前目录下创建
- 创建成功后刷新当前目录列表
- 保持当前目录不变

#### 新建 Markdown 文件

- 在当前目录下创建 `.md` 文件
- 创建成功后刷新当前目录列表
- 之后进入与“点击其他 Markdown 文件”一致的打开决策流程
- 不直接绕过用户选择自动切换

### 4.9 文件管理栏显隐入口

显隐入口分为两层：

1. 即时开关
   - 放在主窗口的 View 菜单中
   - 作为主窗口级能力，不绑定编辑器内部工具栏
2. 默认配置开关
   - 放在设置页“视图”分组
   - 与当前“大纲默认显示”配置并列

此外，当文件管理栏关闭后，左侧应保留一个轻量唤起手柄，方便用户无需回菜单即可重新打开。

## 5. renderer 架构设计

### 5.1 新增主窗口级侧栏容器

在 [HomeView](../../../wj-markdown-editor-web/src/views/HomeView.vue) 新增主窗口外层 split 布局：

- 新增 `FileManagerPanel` 组件
- 新增主窗口侧栏 gutter
- 保留现有 [LayoutTop](../../../wj-markdown-editor-web/src/components/layout/LayoutTop.vue)、[LayoutMenu](../../../wj-markdown-editor-web/src/components/layout/LayoutMenu.vue)、[LayoutContainer](../../../wj-markdown-editor-web/src/components/layout/LayoutContainer.vue) 的角色不变

### 5.2 文件管理栏 renderer 状态

renderer 侧单独维护文件管理栏状态，不混入 `documentSessionSnapshot`：

- 是否显示
- 当前目录路径
- 当前目录列表
- 是否加载中
- 当前目录是否可创建
- 当前目录是否为空状态

当前文件路径仍以 `store.documentSessionSnapshot` 为唯一真相来源，用于高亮与默认目录判断。

### 5.3 打开决策控制器

新增统一的 renderer 打开决策控制器，供以下入口共用：

- 左侧文件管理栏点击 Markdown 文件
- 顶部菜单“最近历史”

该控制器只负责：

- 弹出打开方式选择
- 在当前窗口打开前弹出保存决策
- 调用主进程对应命令

该控制器不直接操作会话状态，不在 renderer 自己拼保存与切换状态机。

## 6. 主进程架构设计

### 6.1 新增目录服务

Electron 侧新增目录服务，负责：

- 读取当前目录一层条目
- 创建文件夹
- 创建 Markdown 文件
- 监听当前目录一层变化

目录读取建议使用 `readdir(..., { withFileTypes: true })`，避免每个条目额外 `stat`。

### 6.2 目录监听策略

目录监听只监听“当前显示目录”，不递归。

实现策略：

- 使用 `fs.watch` 监听当前目录
- 收到目录事件后做短暂防抖
- 防抖结束后直接整目录重扫一层内容

不依赖 `fs.watch` 的事件类型做精细增量 patch，原因是该 API 在不同平台上的事件语义和 `filename` 可用性都不完全稳定。

### 6.3 新增当前窗口切换命令

新增独立 runtime 命令：

- `document.open-path-in-current-window`

该命令必须与现有 `document.open-path` 分离：

- `document.open-path` 保持“新建窗口或聚焦已有窗口”的既定语义不变
- `document.open-path-in-current-window` 专门负责在当前窗口内部切换文档会话

## 7. 当前窗口切换设计

### 7.1 总原则

当前窗口切换文档时，不能直接在原 session 上替换文件内容与路径。

必须采用：

- 新建目标文件对应的新 session
- 将当前窗口重新绑定到新 session
- 停止旧 watcher
- 启动新 watcher
- 发布新 snapshot
- 清理旧 session

不允许采用“沿用原 sessionId，把文件身份直接替换掉”的做法。

### 7.2 不复用原 `sessionId` 的原因

1. `sessionId` 当前表示文档会话身份，不应在同一 `sessionId` 下突然切换为另一份文件。
2. 资源菜单、外部文件变更、滚动恢复等链路已经依赖 `sessionId + revision` 判定上下文是否过期。
3. 直接替换同一 `sessionId` 的文件身份，最容易让旧上下文误命中新文档。

### 7.3 当前窗口切换顺序

1. 校验目标路径
   - 存在
   - 是常规文件
   - 是 `.md`
   - 解析成稳定绝对路径
2. 判断是否为当前文档
   - 若相同则返回 `no-op`
3. 判断目标文件是否已在其他窗口打开
   - 若已打开，则不在当前窗口重复绑定
   - 直接定位已有窗口并返回结构化结果
4. 如用户要求先保存当前文档，则先走统一保存链路
   - 保存成功后继续
   - 保存失败或取消则中止切换
5. 停止当前窗口旧 watcher
6. 读取目标文件并创建新 session
7. 将当前窗口重新绑定到新 session
8. 广播新 snapshot
9. 启动新 watcher
10. 清理旧 session

### 7.4 一文件一窗口原则

若目标 Markdown 已在其他窗口打开，则：

- 当前窗口打开分支不重复创建第二份活动会话
- 直接聚焦已有窗口
- renderer 侧给出统一提示

此行为与当前项目已有的“同一文档只对应一个窗口”的语义保持一致。

## 8. 最近历史复用设计

顶部菜单“最近历史”不再直接调用当前的 `requestDocumentOpenPath()`。

它改为复用统一打开决策控制器：

1. 先选择“当前窗口 / 新窗口 / 取消”
2. 选择“新窗口”时走既有 `document.open-path`
3. 选择“当前窗口”时走新增 `document.open-path-in-current-window`
4. 当前文档未保存时复用相同的保存决策弹框

这样可以保证：

- 左侧文件管理栏与顶部最近历史行为完全一致
- 避免一个入口允许当前窗口打开、另一个入口只会新建窗口的行为分裂

## 9. 主题与国际化

### 9.1 主题

文件管理栏样式必须遵守当前全局主题系统：

- 不写死亮色背景
- 复用现有全局主题变量
- 当前文件高亮、hover、边框、空状态均随主题切换

### 9.2 国际化

以下文案全部进入 i18n：

- 文件管理栏标题
- 新建文件夹
- 新建 Markdown
- 当前窗口打开
- 新窗口打开
- 先保存再切换
- 不保存直接切换
- 不支持的文件类型
- 空状态文案
- 已在其他窗口打开提示

## 10. 异常与边界处理

- 目标路径不存在：返回结构化失败结果，不继续切换
- 目标不是 `.md`：提示“不支持的文件类型”
- 目标文件已在其他窗口打开：聚焦已有窗口，不重复绑定当前窗口
- 当前窗口切换前保存失败：不继续切换
- 当前窗口切换前保存取消：不继续切换
- `recent-missing` 父目录不存在：文件管理栏空状态，不额外展示提示文案
- 点击当前文件：直接 `no-op`

2. 当前文档已保存时，可直接切换并正确重绑 session

8. 切换后旧 watcher 停止，新 watcher 正确绑定

2. 目录变化后通过防抖重扫刷新列表
3. 新建文件夹成功后刷新列表
4. 新建 Markdown 成功后刷新列表
5. `recent-missing` 父目录不存在时返回空状态

## 12. 实施顺序建议

1. 在主窗口壳层引入左侧文件管理栏与外层 split 布局
2. 完成 renderer 侧文件管理栏展示、空状态与基本交互
3. 完成 Electron 侧目录服务与当前目录监听
4. 新增 `document.open-path-in-current-window` 命令
5. 打通当前窗口切换的 session 重绑链路
6. 让顶部最近历史复用统一打开决策控制器
7. 补齐测试、国际化与主题细节

## 13. 最终结论

本功能推荐采用“主窗口左侧文件管理栏 + 主进程目录服务 + 新增当前窗口切换命令”的方案。

该方案的核心收益是：

- 与现有主窗口层级匹配，避免继续膨胀编辑器内部布局
- 与当前 `document-session` 架构一致，不绕过主进程会话真相
- 能同时覆盖左侧文件管理栏与顶部最近历史两条入口
- 能在不破坏现有 `document.open-path` 语义的前提下，安全支持“当前窗口打开”
